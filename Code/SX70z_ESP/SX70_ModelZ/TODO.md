# TODO: 闪光灯检测与拍摄模式分离

## 背景

当前 `shut_mode` 由 `control_task` 根据闪光灯 S2 引脚电平和 menu 值混合决定：

- 有闪光灯 → `shut_mode = '0'`（闪光模式），覆盖 menu 设置
- 无闪光灯 → 根据 menu 设置快门模式（B/T/普通）

这种耦合导致的问题是：**有闪光灯时，无论 menu 选什么模式，shutter_task 都走闪光灯曝光路径**。后续需要将"闪光灯有无"与"拍摄模式"彻底解耦，让每个组合都有明确的预期行为。

## 拍摄模式定义

| menu | 模式 | 说明 |
|------|------|------|
| 0 | AUTO | 测光自动计算快门速度 |
| 1 | BULB (B) | 按住 S1T 时快门打开，松开关闭 |
| 2 | TIME (T) | 按一下快门打开，再按关闭 |
| 3 | MANUAL (M) | 用户手动指定快门速度 |
| 10 | SELF-TIMER | 自拍定时（暂不展开） |

## 预期响应矩阵

> 以下是 **当前代码实际行为**（重构前的 baseline）。每个格子描述对应的曝光流程。

| 拍摄模式 \\ 闪光灯状态 | 无闪光灯 | 有闪光灯（S2 拉低） |
|---|---|---|
| **AUTO (0)** | 测光 → EV → 自动选快门速度 → `mode='1'` 普通曝光（按计算值延时） | `shut_mode` 被强制为 `'0'`，快门速度固定为 index 13（1/30s）→ **闪光灯曝光流程**：光圈打开 → 快门开 → 等 47ms → 触发闪光(FF_PIN) → 剩余延时 → 快门关 → 光圈关 |
| **BULB (1)** | `shut_mode='B'` → 快门开 → 15ms 延迟 → **等待 S1T 释放** → 快门关 | **同样被强制为 `'0'`**，走闪光灯曝光流程，但 shutter_speed 保留用户 M 档设置的值或初始值 |
| **TIME (2)** | `shut_mode='T'` → 快门开 → **等待 S1T 释放 → 等待再次按下** → 快门关 | **同样被强制为 `'0'`**，走闪光灯曝光流程 |
| **MANUAL (3)** | `shut_mode='1'` → 用户指定快门速度 → 普通曝光 | **同样被强制为 `'0'`**，走闪光灯曝光流程，快门速度用用户指定的值（但 gap = shutter_delay - 470，可能为负） |
| **SELF-TIMER (10)** | shutter_task 直接跳过（continue），不拍摄 | 同左（shutter_task 跳过） |

> ⚠️ **核心问题**：有闪光灯时无论 menu 选 BULB 还是 TIME，`shut_mode` 都被 `control_task` 覆写为 `'0'`，导致 B 门/T 门完全失效。MANUAL 模式下插闪光灯也会走 flash 的 gap 逻辑而非普通曝光。

## 重构目标

1. `shut_mode` 不再由 `control_task` 根据闪光灯状态覆写，而是仅由拍摄模式（menu）决定
2. 闪光灯状态（S2 电平）作为独立的上下文传递给 `shutter_task`，由它决定曝光流程中是否执行闪光灯相关步骤（如 sol2_engage/disengage、FF_PIN 触发等）
3. 消除 `control_task` 中 flash_debounce → shut_mode 的耦合路径
